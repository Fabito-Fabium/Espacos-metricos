[Federal University of Technology — Paraná] UTFPR-Article

Go to PDF file of this LaTeX template

This LaTeX template (unofficial) allows the production of articles of the Federal University of Technology — Paraná (UTFPR), including for events such as the Extension and Innovation Seminar (SEI) and the Scientific and Technological Initiation Seminar (SICITE), among others. It was developed using the (default) article LaTeX class. Also, several code snippets developed by users of the TeX-LaTeX Stack Exchange were used.

Status: added by Luiz E. M. Lima, maintenance on demand.
Last updated: October 10, 2023 (Version 1.5).


Este modelo LaTeX (não oficial) permite a produção de artigos da Universidade Tecnológica Federal do Paraná (UTFPR), inclusive para eventos, como o Seminário de Extensão e Inovação (SEI) e o Seminário de Iniciação Científica e Tecnológica (SICITE), entre outros. Foi desenvolvido usando a classe LaTeX artigo (padrão). Além disso, diversos trechos de códigos desenvolvidos por usuários do TeX-LaTeX Stack Exchange foram usados.

1. Arquivos constituintes do projeto

Breve descrição dos arquivos que devem ser manipulados no modelo LaTeX:

  • ./utfpr-article.tex (Principal).
  • ./utfpr-article.bib (Referências — Base de Dados).
  • ./Extras/pre-textual.tex (Elementos Pré-Textuais).
  • ./Extras/post-textual.tex (Elementos Pós-Textuais).

Breve descrição dos demais arquivos constituintes:

  • ./utfpr-article.sty (Pacote de Estilos).
  • ./utfpr-article.pdf (PDF Resultante da Compilação).
  • ./readme.html (Descrição e Informações).
  • ./readme.txt (Breve Descrição).

Os demais arquivos nos diretórios (pastas) do projeto são imagens, carregadas no modelo em formatos suportados, assim como alguns arquivos-fontes para geração das mesmas ou de elementos do documento. Não apagar o conteúdo dos diretórios:

  • ./Extras/.
  • ./Headers/.
  • ./Illustrations/.
  • ./Logos/.
  • ./SGDStamps/.

1.1. Compilação dos arquivos

No diretório do projeto, abra um prompt de comando (no Windows) ou um console (no Linux) e digite make help para obter informações a respeito dos comandos de compilação dos arquivos do modelo LaTeX:

  • ./make.bat (Batchfile para Compilação no Prompt de Comando do Windows).
  • ./makefile (Makefile para Compilação no Console do Linux).

5. Dicas de uso

Sempre que for manipular e/ou substituir um dos arquivos constituintes do projeto, mantenha uma cópia do original num local seguro e/ou renomeie a cópia do original para poder ser seguida como um exemplo no desenvolvimento do seu próprio arquivo.

O único formato de imagens que pode ser inserido ao compilar com LaTeX é o Encapsulated PostScript (EPS). Se for compilar com PDFLaTeX, para produzir um PDF diretamente, tem-se uma escolha mais ampla de formatos de imagens para inserir:

  • JPG, amplamente utilizado na Internet, em câmeras digitais, etc., sendo a melhor escolha para fotos.
  • PNG, um formato sem perdas e muito comum (ainda que não tanto como JPG), sendo a melhor escolha para diagramas (se não for possível gerar uma versão vetorial) e capturas de tela.
  • PDF, é amplamente utilizado para documentos, mas pode ser usado para armazenar imagens também; suporta imagens vetoriais e mapa de bits, mas não é recomendado para o último, pois JPG ou PNG irá fornecer o mesmo resultado com menor espaço em disco.
  • EPS, pode ser usado com ajuda do pacote epstopdf, que só precisa estar instalado; portanto, não há necessidade de carregá-lo no documento.
